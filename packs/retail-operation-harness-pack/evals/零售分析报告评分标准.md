# 零售分析报告质量评分器 (Eval Rubric)

**总分 100 分。** 用于自动评估 Agent 生成的报告是否可正式交付。

### 1. 问题定义（10 分）
- [ ] 明确了业务问题（如：为什么销售下滑）。
- [ ] 明确了分析对象和时间范围。
- [ ] 明确了对比基准（同比、环比或预定目标）。

### 2. 数据可信度（15 分）
- [ ] 说明了分析所用的数据来源及范围。
- [ ] 说明了数据过滤规则（如：已剔除测试订单和全额退款订单）。

### 3. 指标口径（15 分）
- [ ] 对关键指标（销售额、订单数、客单价、复购率等）的计算逻辑无误用。
- [ ] 没有混淆标价（应收）和实付（实收）。

### 4. 分析拆解深度（20 分）
- [ ] 采用了结构化的公式拆解（GMV = 订单数 × 客单价等）。
- [ ] 进行了至少两个维度的下钻（如：用户新老客拆解、商品品类/爆品拆解）。
- [ ] 识别出了主次矛盾（贡献度大小计算准确）。

### 5. 洞察质量（15 分）
- [ ] 洞察不只是表面数字的罗列（不能只说“A下降了所以B下降了”）。
- [ ] 明确区分了相关性和因果性。
- [ ] 注意到了异常节点（如大促后回落、缺货影响）。

### 6. 行动建议可执行性（15 分）
- [ ] 建议具有明确的对象（人群、商品、门店）。
- [ ] 动作明确（降价、触达、上新、补货等）。
- [ ] 给出了后续的验证指标（如何评估该动作有效）。

### 7. 表达与排版（10 分）
- [ ] 结论前置，重点突出。
- [ ] 报告结构符合模板（如高管版要求少讲技术，多讲业务）。

**门槛标准：**
- `≥ 90 分`：高质量，直接交付。
- `80-89 分`：可交付，轻微调整。
- `< 80 分`：必须返工。
