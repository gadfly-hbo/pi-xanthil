## 0. 背景与定位

当前规则记忆模块功能完整（五源注入 + 提议/审批 + 冲突检测 + 使用统计/反馈 + baseline vs memory 评估 + 工作区启用），但**实际从未被用起来**。根因不是功能少，而是**以手工维护为中心**（手写规则/审批提议/勾选启用），与"做分析"主流程割裂，用户无动力维护；且注入是**固定优先级 + 4000 token 静态选择**，与当前任务无关、低优先级源常被压掉。

用户决策：**推倒重来（架构重构 + 范式升级）**，围绕"让记忆成为工作的副产物、低摩擦自动沉淀与召回"。四类痛点全中、四个 2026 趋势能力全要（经验记忆 / 多信号检索 / 治理防漂移 / 可观测闭环）。

**总控已锁定的默认口径（被派发方零自由度，如需调整由用户拍板）**：
1. **relevance 召回**：先用**词法 + 信号混合**（复用 `knowledge-graph.ts` 词汇重叠思路，零新依赖、本地、隐私安全）；embedding 作后续可插拔增强，本轮不引入新依赖。
2. **自动沉淀强度**：**高置信低危自动入库 + 其余进一键复核队列**（对齐 ChatGPT Dreaming / reflective memory，最小化摩擦）。
3. **旧表处理**：**drop & rebuild**（从未投产，无数据迁移）。`metric_definitions` / `business_contexts` **不动**（指标语义层 P1 在建、`BusinessContextPane` 在用），新记忆层以 adapter 当 `fact` 读入。

## 2. 统一记忆模型（契约期写死）

```
MemoryItem {
  id; workspaceId;
  type: "constraint" | "experience" | "episode";   // fact 由 adapter 从既有源投影，不入表
  title; body;
  source: "manual" | "trace" | "derived"; sourceEventIds: string[];
  confidence: number; riskFlags: RiskFlag[];                       // 复用 injection/pii/weak/overbroad
  validFrom; validUntil | null; supersedesId | null;               // 时序与状态演化(防 semantic drift)
  usedCount; lastUsedAt; positiveSignals; negativeSignals; staleAfterDays;  // 折叠原 usage_stats
  scope: "global" | "chat" | "workflow"; enabled; createdAt; updatedAt;
}
MemoryCandidate { ...蒸馏候选，待门禁；含 confidence/riskFlags/sourceEventIds }
RetrievalContext { query; recentMessages?; dataPaths? }            // 多信号检索入参
```

- `constraint`=原 rules；`experience`=新增（trace 沉淀「做法」，并入原 cases）；`episode`=新增（过往 session/flow 摘要）。
- 检索打分：`score = w_rel·relevance(ctx) + w_rec·recency + w_fb·feedback + w_type·typePriority`；治理过滤剔除 suppressed/expired/poison/superseded；top-K 按质量在预算内填充（替代固定优先级硬截断）。
