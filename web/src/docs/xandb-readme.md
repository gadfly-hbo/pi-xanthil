# 数据库模块 操作手册

> **模块定位**：「数据库」（顶部 tab `xan_db`）是 pi-Xanthil 的**外部 / 参考数据中枢**——把分析常用的**环境与对标数据**（人群 / 天气 / 商圈 / 行业 / 竞品）、**本品经营看板**，以及**外部数据库连接**集中到一处，作为日常 / 专题分析的「数据底座补给站」。它提供的是**做分析时要参照的外部事实**，区别于工作区里你自己登记的 draw_data / clean_data。
>
> 📅 更新日期：2026-06-20
>
> 本文按二级 tab 顺序，逐一讲解定位、能力与使用要点。

## 1. the-crowd（`the-crowd`）🚧

**定位**：人群数据库管理。**当前为占位**（即将推出）——规划用于沉淀与管理可复用的人群包 / 人群画像。

---

## 2. 天气（`weather`）

**定位**：基于 **Open-Meteo** 开源天气 API 的中国城市历史天气数据查询。

**用途**：为强天气相关的业务（服饰、生鲜、出行等）补充外部天气因子，辅助归因与季节性分析。数据为公开气象数据，可与销售时序对照看。

---

## 3. 商圈（`business_district`）🚧

**定位**：商圈数据管理。**当前为占位**（即将推出）——规划用于商圈级别的人流 / 业态 / 消费力等参考数据。

---

## 4. 行业（`industry`）

**定位**：由 pi agent **联网检索**生成的**行业大盘情报**——市场规模 / 集中度 / 波特五力 / 趋势等。

**使用要点**
- 输入行业名 + 选模型，点生成；pi agent 联网检索并结构化输出。
- 数值为公开信息估算，**仅供分析参考**，不等同权威统计。
- 数据边界：**只把行业名发给 pi**，不读工作区任何明细。

**用途**：在做业务诊断前，先建立"行业坐标系"（大盘多大、卷不卷、往哪走），让后续结论有外部锚点。

---

## 5. 竞品（`competitor`）

**定位**：由 pi agent **联网检索**生成的**竞争情报**——竞品档案 / 市场份额估计 / 对标矩阵 / 替代风险。

**使用要点**
- 输入品牌 / 竞品名 + 选模型，点生成。
- 份额为公开信息估算，**仅供分析参考**。
- 数据边界：**只把品牌名发给 pi**，不读工作区明细。

**用途**：横向对标——看清自己在竞争格局里的位置与差距，为黄金策 / 行动提供对手视角输入。

---

## 6. 本品（`own_product`）

**定位**：**会员运营 BI 看板**（`BiDashboardPane`）——把已脱敏聚合数据做成可视化经营看板。

**主要能力**
- 配置数据源：选**预设 BI 数据集**（新会员留存 / 老会员召回 slot，经导入对话框上传）或直连 **clean_data** 聚合文件。
- 每张图配置维度 / 度量 / 图型，服务端 / 前端确定性出图。
- 与「汇报版本 → 关联数据集自动出图」共用同一套脱敏聚合数据契约。

**数据边界**：消费的是已脱敏聚合 / clean_data，不碰 draw_data 原始明细。

---

## 7. SQL连接（`sql_connect`）

**定位**：连接**外部数据库**（PostgreSQL / MySQL / MariaDB / SQLite），写 SQL 跑查询，可存为参数化模板。

**主要能力**
- 连接管理：新建 / 编辑 / 删除 / 测试连接（返回延迟），配置加密存于 server 工作区。
- Schema 浏览器：拉取库表结构，点表看列定义。
- SQL 编辑器：支持 `{{param}}` 模板参数（自动提取为输入框）、校验（禁 DDL/DML 误执行）、运行、结果分页 / 排序、导出 CSV。
- Saved Query：把 SQL + 参数存为命名查询挂在连接下；可暴露给 AI 作为「动态参数化数据源」，但**执行结果默认不自动回灌 LLM**。

**安全要点**：生产库强烈建议用**只读账号** + IP 白名单；密码加密存 server。结果要发 LLM 需显式操作。

## 数据安全 · 速查清单

| 操作 | 允许吗？ |
|---|---|
| 行业 / 竞品联网检索（只发行业名 / 品牌名） | ✅ 不读工作区明细 |
| 让行业 / 竞品检索读取本地 draw_data 明细 | ❌ 架构上不发送明细给 pi |
| 本品看板消费已脱敏聚合 / clean_data 出图 | ✅ 确定性出图，LLM 只看 schema |
| 用 SQL 连接的**只读账号**查外部库 | ✅ 推荐 |
| 让 AI 直接连库执行 SQL | ❌ 由用户在面板查询并选择性发送 |
| 把 SQL 查询结果 / 看板明细自动喂给 LLM | ❌ 默认不自动回灌，需显式操作 |

详见 `AGENTS.md §一`。

## 故障排查（FAQ）

- **行业 / 竞品生成失败 / 内容空**：换更稳定的模型；行业名 / 品牌名写具体些；联网检索受网络与公开信息覆盖度影响。
- **天气查不到城市**：确认是中国城市；Open-Meteo 覆盖以经纬度匹配，生僻地名可换就近城市。
- **本品看板无数据**：先在「本品」导入 BI 数据集（留存 / 召回 slot），或登记 clean_data 聚合文件。
- **SQL 连接测试失败**：检查 host 可达 / 端口 / 凭据；PG 看 `pg_hba.conf`，MySQL 看 user@host 授权，SQLite 看文件绝对路径。
- **SQL 校验报 "dangerous statement"**：默认禁 DDL/DML，需改后端白名单。
- **the-crowd / 商圈点开是占位**：这两个子模块即将推出，暂未实装。
